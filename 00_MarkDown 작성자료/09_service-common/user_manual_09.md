# 서비스공통 사용자교육자료

## 개요

K-에듀파인 서비스공통은 K-에듀파인 전체 시스템의 기초 정보를 설정하고 운영 환경 및 권한을 관리하는 핵심 공통 서비스입니다.

### :material-folder: 서비스공통

서비스공통은 NEIS에서 연계된 조직·사용자 정보를 기반으로 시스템 전체의 권한과 운영 환경을 통합 관리합니다.

- **조직 및 사용자관리**: 나이스(NEIS)에 등록된 조직 및 사용자정보를 연계 수신하여 체계적으로 관리
- **권한관리**: 사용자별 화면접근 및 자료처리 권한을 관리
- **코드관리**: 공통코드, 물품코드, 금융기관코드 등 코드를 관리
- **포털관리**: 시스템 운영환경 설정, 게시판, 알림 팝업 등을 관리

### :material-folder: 권한관리

K-에듀파인의 권한은 실명 확인이 가능한 개인 계정에만 부여됩니다. 기관마스터는 총괄관리자와 기관관리자를 지정하는 역할만 하며, 모든 권한 부여·회수 시 공문번호 입력 또는 결재가 필수입니다.

- **기관마스터의 업무권한 배제**: 기관마스터의 업무권한을 배제하며, 총괄관리자 또는 기관관리자를 지정하는 역할로 제한
- **결재를 통한 권한신청**: 모든 권한 부여 및 회수 시 공문번호 입력 또는 결재를 통한 권한신청 의무화. 공문번호 없이 등록된 사용자는 등록일로부터 **3일 경과 후 자동으로 권한이 만료**됨
- **관리자 권한 세분화**: 총괄관리자, 업무권한관리자, 기관관리자 지정을 통한 관리업무 분산
- **과도한 권한 부여 방지**: 권한 인수인계 시 인수자의 소속기관에서 부여한 일반사용자권한만 인계
- **권한신청 및 이상자 관리**: 사용자가 직접 신청하는 [권한신청] 기능 및 부적절한 권한을 회수하는 [권한이상자 관리] 기능 제공

!!! Info "핵심요약"
    - **기관마스터 ID로는 업무를 할 수 없음**. 기관마스터 ID로 총괄관리자 또는 각 기관/학교의 기관관리자를 먼저 지정
    - 권한관리 업무 경감을 위해 업무별 또는 부서별로 **업무권한관리자를 지정**하여 관리
    - 총괄관리자가 기금권한관리자를 지정하고 기금권한관리자가 **기금회계권한 사용자 및 기금회계항목을 부여**
    - 권한 인수인계 시 인계자가 **인수자의 소속기관에서 부여 받은 일반사용자권한만 인계**
    - 정기적으로 **권한이상자 관리**를 통해 사용자에게 과다하게 부여된 권한을 회수 및 관리

관리자 유형에 따라 접근할 수 있는 권한 범위가 구분됩니다.

`기관마스터`
:   NEIS에서 수신된 인증서 및 기관마스터ID로 접속하는 최상위 관리자. 총괄관리자 및 기관관리자만을 지정하며 업무권한은 없음

`총괄관리자`
:   기관마스터로부터 권한을 부여 받아 산하 기관 전체를 관리. 포털관리, 수행업무관리, 사용자그룹관리, 사용자그룹별권한관리, 사용자별권한관리 등을 담당

`기관관리자`
:   기관마스터로부터 권한을 부여 받은 관리자(시도교육청/지원청/직속기관/학교). 수행업무관리, 사용자그룹별권한관리, 사용자별권한관리 등을 담당

`업무권한관리자`
:   총괄관리자로부터 권한을 부여 받아, 자신이 맡은 부서별·업무별·기금 관리자그룹 범위 안에서 일반사용자에게 권한을 부여하는 관리자

관리자 지정은 기관 유형별로 다음과 같은 순서로 이루어집니다. 공문번호 미입력 시 지정일로부터 **3일 후 권한이 자동 만료**됩니다.

| 기관 구분 | 권한 부여 흐름 |
| --------- | -------------- |
| 총괄센터(교육부) | 운영자 ID → (공문번호) → 재정영역/업무관리 총괄센터관리자 |
| 본청, 정보원 | 기관마스터ID → (공문번호) → **시도총괄관리자** 또는 **기관관리자** → (결재/공문번호) → 관리자그룹별담당자관리(부서/기금[재]/업무권한관리자) |
| 교육지원청 | 기관마스터ID → (공문번호) → **지역총괄관리자** 또는 **기관관리자** → (결재/공문번호) → 관리자그룹별담당자관리(부서/기금[재]/업무권한관리자) |
| 학교, 직속기관 등 | 기관마스터ID → (공문번호) → **재정영역 기관관리자** 또는 **업무관리 기관관리자** |

## 세부업무

### :material-folder: 조직 및 사용자관리
NEIS에서 연계 수신된 조직 및 사용자 정보를 K-에듀파인에서 확인하고 관리합니다. 메뉴는 예산·회계 등을 다루는 **재정 영역**과 문서관리 등을 다루는 **업무관리 영역**으로 나뉘며, 각 영역의 메뉴(`(재정)`·`(업무)`)에서 따로 처리합니다.

=== "조직관리"


    - **(업무)조직관리** (1): NEIS에서 수신한 업무 조직정보를 관리. 하위 비공식(가상) 조직 추가 및 기존 조직 정보 수정 가능. **공식 조직의 부서 이동은 나이스 사용자정보 연계를 통해서만 가능**하며 K-에듀파인에서 직접 변경 불가
        { .annotate }

        1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 조직및사용자 → (업무)조직관리 → 조직관리

    - **조직이력관리** (1): 조직 변경 이력을 조회
        { .annotate }

        1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 조직및사용자 → (업무)조직관리 → 조직이력관리

    - **관인관리** (1): 기관 관인 이미지를 등록하고 관리
        { .annotate }

        1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 조직및사용자 → (업무)조직관리 → 관인관리

=== "사용자관리"

    - **(업무)사용자관리** (1): NEIS에서 연계된 사용자 정보를 관리. 겸임 및 파견, 자료권한을 추가하거나 삭제 가능. [부서(학교)사용자관리]로 부서별 사용자를 일괄 확인하며, **부서장은 [부서장지정] 기능으로 부서당 1명만 설정 가능**
        { .annotate }

        1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 조직및사용자 → (업무)사용자관리 → (업무)사용자관리

    - **계정잠금사용자관리** (1): 로그인 실패 횟수 초과 또는 장기 미접속으로 잠긴 계정을 해제. 상태구분 "잠금" 조회 → 대상자 체크 → [잠금해제]. **장기 미접속으로 잠긴 계정은 해제 당일 미접속 시 익일에 자동 재잠금 처리됨**
        { .annotate }

        1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 조직및사용자 → (업무)사용자관리 → 계정잠금사용자관리

    - **사용자부서이동** (1): 사용자를 업무관리 조직 내 하위 비공식(가상) 조직으로 이동. 조직 선택 후 사용자 체크 → [>>] 또는 [<<]로 이동 → [저장]
        { .annotate }

        1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 조직및사용자 → (업무)사용자관리 → 사용자부서이동

    - **감사담당자관리** (1): 감사담당자를 등록하고 열람이력을 관리. 신규 등록 시 [행추가]로 사용자를 추가한 후 열람조직, 열람기간, 열람사유를 입력하여 저장
        { .annotate }

        1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 조직및사용자 → (업무)사용자관리 → 감사담당자관리

### :material-folder: 홍보문구 관리
표준서식의 상단 또는 하단에 표시되는 홍보문구를 등록하고 관리합니다. 교육청 전체에 적용되는 **교육청홍보문구**와 개별 기관에서 운영하는 **기관홍보문구**로 구분되며, 모두 [조직관리] 화면 오른쪽 상단의 버튼에서 설정합니다.

:material-sitemap-outline: 서비스공통 → 조직및사용자 → (업무)조직관리 → 조직관리

!!! Tip "교육청홍보문구 설정"
    **시도총괄관리자(업무)** 권한을 가진 사용자가 교육청홍보문구를 작성·설정함

    1. 조직관리 화면 오른쪽 상단의 [교육청홍보문구] 버튼 클릭
    2. [신규] 선택 후 홍보문구명, 홍보문구위치, 기간, 홍보문구내용 입력
    3. [저장]으로 시스템에 반영하며, [조회]로 등록 목록 확인, [삭제]로 불필요한 문구 제거

!!! Tip "기관홍보문구 설정"
    **기관관리자(업무)** 권한을 가진 사용자가 기관 자체 홍보문구를 작성·수정함

    1. 조직관리 화면 오른쪽 상단의 [홍보문구] 버튼 클릭
    2. [신규] 선택 후 홍보문구명, 홍보문구위치, 기본홍보문구여부, 일괄지정여부, 홍보문구내용 입력
    3. [저장]으로 시스템에 반영하며, [조회]로 등록 목록 확인, [삭제]로 불필요한 문구 제거

!!! Warning "일괄지정 제약사항"
    일괄지정은 최초 등록 시에만 적용되며, 일괄지정으로 등록한 홍보문구의 일괄 수정·삭제 불가

상위기관에서 설정한 홍보문구가 하위 기관에 표시되는 경우, 해당 기관의 기관관리자(업무)가 [조직관리] 화면에서 기관을 선택하고 [홍보문구] 버튼을 클릭한 뒤 사용할 문구의 **기본홍보문구**를 '예'로 변경하여 저장하면 소속 기관만의 문구로 운영할 수 있습니다.

??? Info "기관 홍보문구가 적용되지 않을 때"
    - 기본홍보문구 변경 후에도 적용되지 않으면 **우선적용여부**가 '교육청 홍보문구'로 설정되어 있는지 확인
    - 우선적용여부가 '교육청 홍보문구'이면 교육청에서 작성한 교육청홍보문구가 우선 적용됨
    - 상위기관이 [기본홍보문구: 예], [일괄지정여부: 하위부서 또는 하위기관및부서]로 등록한 경우 소속 조직의 기본홍보문구가 변경됨

??? Info "홍보문구와 홍보요청 구분"
    - 표준서식에 표시할 문구를 만드는 작업은 위의 **홍보문구** 기능을 사용
    - 특정 행사(입학설명회 등)에 대한 홍보를 요청하려는 경우는 [업무관리] > [업무지원] > [홍보관리] > [홍보요청] 메뉴 이용

### :material-folder: 관련조직 설정
2개 이상의 학교를 묶어 대표 학교의 단일 문서함을 함께 사용하도록 설정하는 기능입니다. **시도총괄관리자(업무)** 권한을 가진 사용자가 [조직관리] 화면에서 처리합니다.

:material-sitemap-outline: 서비스공통 → 조직및사용자 → (업무)조직관리 → 조직관리

!!! Tip "관련조직 설정 절차"
    1. 대표 학교로 설정할 학교를 조회한 뒤 하위조직목록에서 선택하고 [관련조직설정] 버튼 클릭
    2. 관련조직설정 팝업에서 [관련조직추가]로 비대표 학교를 추가하고 [저장]
    3. 팝업을 닫고 동일 메뉴에서 비대표 학교를 조회하여 하위조직목록에서 선택, [조직수정] 버튼 클릭
    4. 조직수정 팝업의 상위조직코드를 [돋보기]로 대표 학교의 조직코드로 변경 후 [저장]

!!! Warning "처리 순서 주의"
    반드시 **관련조직설정 → 비대표 학교의 상위조직코드 변경** 순서로 처리. 순서를 거꾸로 하면 비대표 학교가 공식 조직 기능을 그대로 유지함

저장이 완료되면 비대표 학교는 공식 조직 기능을 상실하고 대표 학교의 문서함을 사용하게 되며, 조직구성도에서 대표 학교 하위에 위치하고 조직명 앞의 '문' 표기가 사라집니다.

### :material-folder: 권한관리
기관마스터 계정으로 관리자를 지정하는 것을 시작으로, 총괄관리자 또는 기관관리자가 일반사용자 권한을 부여하는 흐름으로 진행됩니다.

#### 총괄/기관관리자 지정
기관마스터 계정으로 접속하여 총괄관리자 및 기관관리자를 지정하며, 기관마스터는 이 지정 역할만 수행합니다. 총괄/기관관리자그룹별담당자관리 (1) 에서 관리자그룹 선택 → [행추가]로 담당자 추가 → 공문번호 입력 → [저장] 순서로 처리합니다. **공문번호 미입력 시 3일 후 권한이 자동 만료**됩니다.
{ .annotate }

1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → 관리자그룹관리 → 총괄/기관관리자그룹별담당자관리

#### 일반사용자 권한 부여
총괄관리자 또는 기관관리자가 수행업무 기반의 사용자그룹을 통해 권한을 부여합니다.

- **수행업무관리** (1): 기관별 필요한 수행업무를 정의. 세부시스템 선택 후 [행추가]로 수행업무를 등록하고, 메뉴목록조회 팝업에서 해당 수행업무에 필요한 메뉴를 추가
    { .annotate }

    1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → (업무)사용자그룹관리 → 수행업무관리

- **사용자그룹관리** (1): 수행업무를 묶어 사용자그룹을 구성. 생성한 사용자그룹에 수행업무목록 팝업으로 수행업무를 연결. **기본여부를 "예"로 설정 시** 사용자 부서이동/겸임/파견 등에 기본권한으로 자동 부여됨
    { .annotate }

    1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → (업무)사용자그룹관리 → 사용자그룹관리

- **사용자그룹별권한관리** (1): 특정 사용자그룹에 여러 사용자를 일괄 등록할 때 사용. [행추가]로 사용자 추가, 자료권한 일괄등록 팝업으로 다중 조직을 한 번에 등록 가능
    { .annotate }

    1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → (업무)접근권한관리 → 사용자그룹별권한관리

- **사용자별권한관리** (1): 특정 사용자에게 여러 사용자그룹 권한을 개별 부여. 사용자 조회 후 [행추가]로 사용자그룹을 추가하고 자료권한 조직을 연결. **공문번호 미입력 시 3일 후 권한 자동 만료**
    { .annotate }

    1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → (업무)접근권한관리 → 사용자별권한관리

#### 업무권한관리자 지정 및 권한 부여
총괄관리자가 부서/업무/기금 업무권한관리자를 지정하고, 지정된 업무권한관리자가 해당 그룹의 사용자에게 권한을 위임 부여하는 구조입니다.

- **관리자그룹별담당자관리** (1): 총괄관리자가 부서/업무/기금 업무권한관리자를 지정. 관리자그룹 선택 후 [행추가]로 관리자를 추가하고 공문번호 입력
    { .annotate }

    1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → (재정/업무)업무권한관리자그룹관리 → 관리자그룹별담당자관리

- **업무권한관리자별 사용자관리** (1): 업무권한관리자가 자신에게 부여된 그룹 범위 내의 사용자그룹을 일반사용자에게 부여. **본인이 부여 받은 관리자그룹별 사용자그룹에 대해서만 권한 부여 가능**. 기금회계권한인 경우 자료권한 선택 후 기금회계 항목까지 추가
    { .annotate }

    1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → (업무)접근권한관리 → 업무권한관리자별 사용자관리

#### 권한 인수인계
인사 이동 시 전임자 권한을 후임자에게 이전합니다. 권한인수인계 (1) 에서 인수자 먼저 조회 → 인계자 조회 → 인계할 사용자그룹 체크 → 공문번호 입력 → [확정] 순서로 처리합니다.
{ .annotate }

1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → (업무)접근권한관리 → 권한인수인계

!!! Warning "인수인계 범위 제한"
    - 인수자를 먼저 조회해야 인계자 조회 가능
    - 인계자는 인수자와 동일 소속이거나 과거 소속 이력이 있어야 함
    - 타 기관에서 받은 권한 및 관리자(총괄/기관/업무/부서/기금) 권한은 인계 불가

#### 권한이상자 관리
정기적으로 과다 권한을 보유한 사용자를 점검하고 권한을 회수합니다. 모든 권한 회수 시 공문번호 입력이 필수입니다.

- **사용기관이상자관리** (1): 사용자그룹의 사용기관 권한이 없는 사용자를 조회. [대상자 체크]→[대상자 지정]→[저장]→["사용기관규칙위반" 대상자 체크]→[공문번호 입력]→[권한회수]→[저장]
    { .annotate }

    1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → (업무)권한이상자관리 → 사용기관이상자관리

- **허용인원초과자관리** (1): 조직별 허용인원 임계치 초과 사용자를 조회하고 권한 회수
    { .annotate }

    1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → (업무)권한이상자관리 → 허용인원초과자관리

- **권한삭제대상조회** (1): 권한이상자로 지정된 대상자를 일괄 조회하여 권한 회수
    { .annotate }

    1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → (업무)권한이상자관리 → 권한삭제대상조회

#### 권한신청
사용자가 직접 필요한 권한을 결재를 통해 신청하는 기능입니다. 시도총괄관리자가 [시스템 운영설정]에서 권한신청 사용여부를 "사용"으로 설정해야 이용 가능합니다.

- **권한신청(사용자)** (1): 일반사용자가 직접 권한을 신청. [신규] → 시스템 선택 → [권한추가] 또는 [전임자권한추가]로 사용자그룹 선택 → 자료권한 조직 등록 → [결재요청] → [결재올림]. **[전임자권한추가]** 기능으로 전임자가 현 소속에서 부여 받은 권한을 확인하고 선택적으로 신청 가능
    { .annotate }

    1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 개인설정 → 권한신청(사용자)

- **권한신청(관리자)** (1): 관리자가 사용자 권한을 대리 신청. 대상자 목록에 사용자를 먼저 추가한 후 사용자 신청과 동일한 절차로 진행
    { .annotate }

    1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 권한관리 → (업무)접근권한관리 → 권한신청(관리자)

!!! Warning "권한신청 주의사항"
    - 전결 처리 시 권한이 자동 부여되지 않음 (결재 완료 후 부여)
    - 결재상태가 "임시저장"인 경우에만 수정 가능
    - 이미 보유한 사용자그룹은 중복 추가 불가
    - 시도총괄관리자(업무)가 서비스공통 > 포털관리 > 운영환경관리 > (업무)시스템 운영설정에서 권한신청 사용 여부를 '사용'으로 설정한 경우 업무관리 영역으로 권한신청(사용자, 관리자) 사용 가능

### :material-folder: 포털관리

        { .annotate }

        { .annotate }

        1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 포털관리 → 운영환경관리 → (업무)시스템 운영설정

        1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 포털관리 → 포털관리 → 알림정보

    - **설문정보관리** (1): 교육청 설문조사 생성 및 결과 관리. 설문진행상태가 "설문진행중"이어야 일반사용자가 참여 가능
        { .annotate }

        1. :man_raising_hand: :material-sitemap-outline: 서비스공통 → 포털관리 → 포털관리 → 설문정보관리

## 주요 FAQ
사용자 지원 과정에서 자주 접수되는 서비스공통 관련 질문과 답변입니다.

!!! question "타시도로 전출, 퇴직한 선생님이 사용자관리에 조회됩니다. 어떻게 삭제해야 하나요?"
    K-에듀파인은 나이스(NEIS)로부터 사용자정보 연계를 수신하여 사용자 계정이 관리되며, 개인근무사항정보 연계를 수신하여 재직상태가 관리됨.

    - **타시도로 전출된 경우**: 나이스에서 해당 선생님 계정의 사용가능여부가 **'아니요'**로 설정된 상태에서 나이스 인사담당자가 사용자정보를 재연계 처리해야 함.
    - **퇴직한 경우**: 나이스에서 해당 선생님의 재직상태가 **'퇴직'**으로 설정된 상태에서 개인근무사항정보를 재연계 처리하거나, 계정의 사용가능여부가 **'아니요'**로 설정된 상태에서 나이스 인사담당자가 사용자정보를 재연계 처리해야 함.

!!! question "퇴직한 사용자는 어떻게 조회하나요?"
    K-에듀파인은 나이스(NEIS)의 개인근무사항정보 연계를 수신하여 재직상태를 관리하며, 재직상태가 **'퇴직'**이면 해당 계정의 사용가능여부가 **'아니요'**로 변경됨. 따라서 [서비스공통] > [조직및사용자] > [(업무)사용자관리] > [사용자관리] 화면에서 퇴직자를 조회하려면 조회조건의 **'미사용여부 포함'**을 체크하여 조회하면 됨.

!!! question "나이스에서는 퇴직 처리했는데 K-에듀파인에서 계속 조회됩니다."
    K-에듀파인은 나이스(NEIS)의 개인근무사항정보 연계를 수신하여 재직상태를 관리하며, 재직상태가 '퇴직'이면 해당 계정의 사용가능여부가 '아니요'로 변경됨.
    따라서 나이스에서 해당 사용자의 재직상태를 확인한 뒤 개인근무사항정보 연계를 재송신하면 K-에듀파인에 반영되어 조회되지 않음. 또는 나이스에 문의하여 해당 계정의 사용가능여부를 '아니요'로 설정한 뒤 사용자정보 연계를 송신해도 해당 사용자가 조회되지 않음.

!!! question "표준서식에 들어가면 '사용자 정보를 구할 수 없습니다'라고 나옵니다. 어떻게 해야 하나요?"
    사용자 계정이 미사용 처리되었거나 사용기간이 유효하지 않은 경우, 또는 소속조직이 일치하지 않는 경우 조회되지 않을 수 있음.
    K-에듀파인은 나이스(NEIS)에서 사용자정보 연계를 수신하므로, 나이스에서 계정의 사용가능여부, 사용기간(시작일자·종료일자), 소속조직을 확인한 뒤 K-에듀파인으로 연계 송신해야 함.
    또한 기관관리자(업무)에게 요청하여 [서비스공통] > [조직및사용자] > [(업무)사용자관리] > [사용자관리] 메뉴에서 사용자ID를 클릭, 사용자정보조회 팝업의 **'업무사용자'** 항목이 **'예'**로 설정되어 있어야 결재경로에서 조회됨.

!!! question "학교명이 변경됐는데 공람처리 시 조직도에 이전 교명이 그대로 표시됩니다."
    K-에듀파인은 나이스(NEIS)로부터 조직정보 연계를 수신하여 공식 조직을 관리하므로, 변경된 학교명이 반영되지 않으면 먼저 나이스에서 변경된 교명으로 조직정보를 연계 송신해야 함.
    나이스 송신 후에도 이전 교명으로 조회된다면, 시도총괄관리자(업무)에게 문의하여 연계 설정(나이스 조직정보 수신이 '수동'으로 설정된 경우)을 확인한 뒤 [서비스공통] > [조직및사용자] > [(업무)수신관리] > [조직수신관리] 메뉴에서 연계 수신 내역을 [반영] 요청해야 함.

!!! question "교육청 홍보문구는 누가, 어떻게 작성하나요?"
    시도교육청의 시도총괄관리자(업무) 권한을 가진 사용자가 [서비스공통] > [조직및사용자] > [(업무)조직관리] > [조직관리] 화면 오른쪽 상단의 [교육청홍보문구] 버튼을 통해 작성함.
    [신규]로 홍보문구명·홍보문구위치·기간·홍보문구내용을 입력하고 [저장]하여 등록함.

!!! question "기관에서 자체 홍보문구를 만들려면 어떻게 해야 하나요?"
    기관관리자(업무) 권한을 가진 사용자가 [서비스공통] > [조직및사용자] > [(업무)조직관리] > [조직관리] 화면 오른쪽 상단의 [홍보문구] 버튼에서 [신규]로 홍보문구명·홍보문구위치·기본홍보문구여부·일괄지정여부·홍보문구내용을 입력하고 저장하며, 메뉴가 보이지 않는다면 권한 부여 여부를 먼저 확인해야 함.
    단, 일괄지정은 최초 등록 시에만 적용되며 이후 일괄 수정·삭제는 불가능함.

!!! question "상위기관 영향으로 보이는 홍보문구를 우리 기관 문구로 바꾸려면 어떻게 하나요?"
    해당 기관의 기관관리자(업무)가 [서비스공통] > [조직및사용자] > [(업무)조직관리] > [조직관리] 화면에서 기관을 선택하고 [홍보문구] 버튼을 클릭한 뒤 사용할 문구의 기본홍보문구를 '예'로 변경하여 소속 기관 문구로 설정함.
    변경 후에도 적용되지 않으면 우선적용여부가 '교육청 홍보문구'로 설정되어 있는지 확인해야 하며, 이 경우 교육청에서 작성한 교육청홍보문구가 우선 적용됨.

!!! question "관련조직설정은 어떻게 하나요?"
    2개 이상의 학교를 묶어 대표 학교의 단일 문서함을 함께 사용하기 위한 기능으로, 시도총괄관리자(업무)가 [서비스공통] > [조직및사용자] > [(업무)조직관리] > [조직관리] 화면에서 처리함.
    대표 학교를 선택해 [관련조직설정] 버튼으로 비대표 학교를 추가·저장한 뒤, 비대표 학교를 [조직수정]하여 상위조직코드를 대표 학교의 코드로 변경함.

    !!! Warning "순서 준수"
        반드시 **관련조직설정 → 상위조직코드 변경** 순서로 처리해야 하며, 순서가 바뀌면 비대표 학교가 공식 조직 기능을 그대로 유지하게 됨.

!!! question "파견자가 원소속 기관의 문서를 열람하려면 어떻게 해야 하나요?"
    파견자가 원소속 기관의 문서를 열람하려면 원소속 기관관리자(업무)로부터 자료권한을 신청·승인 받아야 함.
    원소속 기관관리자(업무)가 [서비스공통] > [조직및사용자] > [(업무)사용자관리] > [사용자관리] 메뉴에서 해당 파견자를 선택한 뒤, [자료권한 신청]으로 원소속 자료권한을 신청하고 [자료권한 신청승인]에서 [승인] 처리하면 됨. 자료권한 승인 시 기본권한이 자동 부여됨.
    파견자는 승인 후 K-에듀파인에 재로그인해야 원소속 문서를 열람할 수 있으며, 이때 인터넷 창을 닫는 것이 아니라 반드시 **로그아웃 버튼을 클릭한 뒤 재로그인**해야 함.

!!! question "감사담당자 권한은 어떻게 부여하나요?"
    감사담당자 권한은 다음 두 가지 방법으로 부여함.

    - 감사담당자: 기관관리자(업무)가 [서비스공통] > [권한관리] > [(업무)접근권한관리] > [사용자별권한관리] 화면에서 사용자를 조회한 뒤, 사용자별 사용자그룹의 [행추가]로 세부시스템 '문서관리'의 '감사담당자' 권한을 부여함.
    - 감사열람조직: 총괄관리자(업무)가 [서비스공통] > [조직및사용자] > [(업무)사용자관리] > [감사담당자관리] 화면에서 [신규]로 사용자를 추가하고 열람조직·열람기간·열람사유를 입력한 뒤 [저장]하면, 해당 사용자가 감사담당자로 지정되어 열람기간 동안 열람조직의 문서를 열람할 수 있음.

!!! question "장기 미접속으로 잠긴 계정을 해제했는데 계속 다시 잠깁니다."
    장기 미접속은 사용자가 마지막으로 로그인한 일자를 기준으로, 시도에 설정된 장기미접속자 판단 기준일에 따라 관리됨.
    계정 잠금 해제는 기관 내 기관관리자(재정/업무)가 [서비스공통] > [조직및사용자] > [(재정/업무)사용자관리] > [계정잠금사용자관리] 화면에서 처리함.
    단, **계정 잠금 해제일에 K-에듀파인에 접속하지 않으면 익일에 다시 잠금 처리**되므로, 반드시 해제일 당일에 K-에듀파인에 접속해야 함.

!!! question "겸임자·파견자가 인사이동자조회에서 조회되지 않아요."
    인사이동자조회 화면은 신규 사용자 또는 부서이동(원소속 변경)으로 전입된 사용자를 확인하는 화면임.
    겸임 및 파견 사용자는 인사이동자조회가 아닌 [사용자관리] 화면에서 확인할 수 있음.

!!! question "폐기한 관인을 다시 사용할 수 있나요?"
    폐기 처리한 관인은 상태를 다시 **'사용'**으로 변경할 수 없음. 실수로 폐기했더라도 기존 관인을 되살릴 수 없으므로, [서비스공통] > [조직및사용자] > [(업무)조직관리] > [관인관리] 메뉴에서 **[신규]** 기능으로 관인을 등록한 뒤 사용해야 함.

!!! question "휴직자가 복직했는데 K-에듀파인 권한이 반영되지 않아요."
    휴직자가 복직하여 K-에듀파인을 다시 사용하려면 다음 절차로 권한을 재부여해야 함.

    - 기관관리자(업무)가 [서비스공통] > [권한관리] > [(업무)접근권한관리] > [사용자별권한관리] 메뉴에서 복직자를 조회한 뒤 **'업무관리휴직자권한'**을 회수하고, [기본권한 일괄등록] 기능으로 업무관리 기본권한을 부여함.
    - 기관관리자(재정)가 [서비스공통] > [권한관리] > [(업무)접근권한관리] > [사용자별권한관리] 메뉴에서 복직자를 조회한 뒤 [기본권한 일괄등록] 기능으로 재정 기본권한을 부여함.
    - 장기 미접속으로 계정이 잠금 처리된 경우, 기관관리자가 [서비스공통] > [조직및사용자] > [(재정/업무)사용자관리] > [계정잠금사용자관리] 메뉴에서 복직자를 조회·체크한 뒤 [잠금해제] 기능으로 계정 잠금을 해제함.

!!! question "퇴직자를 조직도에서 보이지 않게 하려면 어떻게 해야 하나요?"
    K-에듀파인은 나이스(NEIS)로부터 인사정보 연계를 수신하여 사용자 계정을 관리하므로, 나이스에서 해당 사용자의 재직상태를 **'퇴직'**으로 설정하여 K-에듀파인으로 연계 전송하면 퇴직자 계정의 사용가능여부가 **'아니요'**로 자동 처리되어 조직도에서 조회되지 않음.

!!! question "겸임, 자료권한 표식의 화살표는 무엇을 의미하나요?"
    겸임 표식은 나이스(NEIS)로부터 겸임정보 연계를 수신하여 겸임 처리된 사용자임을 식별하기 위한 표식이며, 자료권한 표식은 [서비스공통] > [조직및사용자] > [(업무)사용자관리] > [사용자관리] 메뉴에서 자료권한 신청 및 승인을 받은 사용자임을 식별하기 위한 표식임.
    표식의 화살표가 **우측 상단**을 향하면 현재 선택된 소속조직이 아닌 타 소속으로 겸임·자료권한을 보유한 상태이며, **좌측 하단**을 향하면 현재 선택된 소속조직으로 겸임·자료권한을 보유한 상태임.

!!! question "사용자관리에서 [사용중지] 버튼이 회색으로 비활성화되어 있는데 왜 그런가요?"
    K-에듀파인은 나이스(NEIS)로부터 사용자정보 연계를 수신하여 사용자 계정을 관리하므로, 계정의 사용여부 또한 나이스 사용자정보 연계로 관리됨. 이에 따라 관리자가 K-에듀파인에서 임의로 사용자 계정을 미사용 처리할 수 없도록 [사용중지] 버튼이 비활성화 상태로 제공됨.

!!! question "부서장 지정 및 삭제는 어떤 메뉴에서 어떻게 하나요?"
    부서장 지정·삭제는 권한에 따라 다음 두 가지 방법으로 처리함.

    - **기관관리자(업무)**: [서비스공통] > [조직및사용자] > [(업무)사용자관리] > [사용자관리] 화면에서 부서장으로 지정할 사용자를 선택한 뒤 [부서장지정] 버튼을 클릭하며, 삭제 시에는 부서장을 선택한 뒤 [부서장삭제] 버튼을 클릭함.
    - **처리과문서담당자**: [개인설정] > [부서(학교)사용자관리] 화면에서 부서장으로 지정할 사용자를 선택한 뒤 [부서장지정] 버튼을 클릭하며, 삭제 시에는 부서장을 선택한 뒤 [부서장삭제] 버튼을 클릭함.

!!! question "자료권한 신청 및 승인은 어떻게 하나요?"
    자료권한은 원소속 기관관리자(업무)가 신청하고, 자료권한 대상 조직의 기관관리자(업무)가 결재(승인)하여 부여됨.

    - **신청**: 원소속 기관관리자(업무)가 [서비스공통] > [조직및사용자] > [(업무)사용자관리] > [사용자관리] 화면에서 사용자를 선택하고 [자료권한 신청] 버튼을 클릭하여 신청정보를 입력한 뒤 [저장]함. 선택한 사용 시스템에 따라 기본권한이 함께 부여됨.
    - **승인**: 자료권한 신청을 받은 부서의 기관관리자(업무)가 동일 화면에서 [자료권한 신청승인] 버튼을 클릭하고, 신청 상태의 사용자를 체크한 뒤 [승인]하면 권한이 부여됨.

!!! question "다른 부서의 업무를 해야 하는데 어떻게 해야 하나요?"
    다른 부서 소속으로 업무를 진행하려면 나이스(NEIS) 겸임정보 연계를 통한 겸임 처리를 받거나, K-에듀파인에서 겸임과 동일한 기능인 자료권한 신청 및 승인을 받아야 함.
    자료권한 신청·승인 절차는 위 '자료권한 신청 및 승인' 안내와 동일하며, 원소속 기관관리자(업무)가 [자료권한 신청]한 뒤 대상 부서의 기관관리자(업무)가 [자료권한 신청승인]하면 됨.

!!! question "겸임 설정은 어떻게 하나요?"
    겸임은 나이스(NEIS)로부터 겸임정보 연계를 수신하여 관리됨. 따라서 K-에듀파인에서 직접 설정할 수 없으며, 관할청의 나이스 인사담당자에게 요청하여 겸임 처리해야 함.

!!! question "자료권한을 중지하거나 회수하려면 어떻게 해야 하나요?"
    자료권한 대상 조직의 기관관리자(업무)가 [서비스공통] > [조직및사용자] > [(업무)사용자관리] > [사용자관리] 화면에서 [자료권한 신청승인] 버튼을 클릭하여 자료권한을 중지하거나 반려할 수 있음.

    - **승인 전(반려)**: 자료권한 상태를 '신청'으로 조회한 뒤 반려할 사용자를 체크하고 [반려] 버튼을 클릭함.
    - **승인 후(중지)**: 자료권한 상태를 '승인'으로 조회한 뒤 중지할 사용자를 체크하고 [자료권한 중지] 버튼을 클릭함.

!!! question "사립학교인 경우 인사(연계)정보를 재전송하려면 어떻게 하나요?"
    인사정보는 나이스(NEIS)에서 K-에듀파인으로 연계 송신됨. 따라서 K-에듀파인에서는 인사정보 연계 전송을 직접 처리할 수 없으며, 인사정보 재전송은 나이스 시스템에 문의해야 함.
